# Forbidden Language Guard — Behavior Spec

> **Implementation:** [`scripts/forbidden-language-guard.js`](scripts/forbidden-language-guard.js)
> **Wired in:** [`hooks.json`](hooks.json) — `PreToolUse` matcher `Write|Edit|MultiEdit`
>
> This document describes the intent and behavior of the hook. The active code Claude Code runs lives in the script linked above. The script enforces a small set of the most clear-cut anti-PLB phrases to avoid false-positive fatigue. The full PLB voice doctrine lives in Collective Wisdom, not in this plugin.

## Purpose

Catch anti-PLB language before it gets written to a file. The user's brand voice is calm warm intelligent authority — words like "hustle," "guru," "crush it," and "passive income" actively contradict that posture.

## How It Works

This hook fires before any file write. The hook script:

1. Reads the content about to be written from the tool input
2. Scans for a curated list of anti-PLB phrases (defined in the script itself)
3. If forbidden language is detected, blocks the write and surfaces the finding
4. The user can override with "language guard false positive, proceed."

## Behavior

**When forbidden language is detected:**

The actual hook output is generated by [`scripts/forbidden-language-guard.js`](scripts/forbidden-language-guard.js) and looks like:

```
⚠️  PLB LANGUAGE GUARD — flagged anti-PLB language

File: [path]

Detected:
  • "[phrase]" — [reason]
    Context: "...[surrounding text]..."

Your options:
  1. Rephrase using your own brand voice.
  2. False positive? Tell Claude: "language guard false positive, proceed."
  3. Cancel the write: tell Claude "cancel."
```

Wait for user response before proceeding.

**When user picks rephrase:**

Suggest a methodology-aligned alternative. Show the before/after diff. Confirm with user. Then write the revised content.

**When user picks override:**

Note the override in MEMORY.md briefly: "Voice override: kept '[word]' in [filename] — user choice." Then write as requested.

**When user picks cancel:**

Abort the write. Wait for further instruction.

## Implementation Note

This hook is implemented as a real Node.js script (see the link at the top of this doc). It is registered in [`hooks.json`](hooks.json) and runs automatically on Write, Edit, and MultiEdit tool calls.

The forbidden phrase list lives inside the script. To extend it, edit [`scripts/forbidden-language-guard.js`](scripts/forbidden-language-guard.js) and bump the plugin version.

## What Counts as a "Write" That Triggers This Hook

- Writing files to brain/ (especially source-of-truth, convictions, learnings)
- Writing files to /mnt/user-data/outputs/
- Writing content that will be shared externally
- Any file that's part of the user's public-facing or brand-aligned content

## What Does NOT Trigger This Hook

- Internal scratchpads
- Code files
- Configuration files
- Quoted content from sources (research, references, sample customer language)

When in doubt, surface and let the user decide.

## Critical Rule

This hook is a guard, not a censor. The user can always override. The point is to surface the conflict so they make a deliberate choice — not to block their agency.
